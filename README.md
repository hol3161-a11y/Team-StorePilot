## ✨ 소개

<p>
  <b>StorePilot</b>는 매장 운영에 필요한 다양한 업무를 효율적으로 관리할 수 있도록 개발한 <b>스마트 매장 관리 플랫폼</b>입니다.
</p>

<p>
  매장의 주요 현황을 한눈에 확인할 수 있는 <b>대시보드</b>를 비롯하여
  <b>매출 관리</b>, <b>메뉴 관리</b>, <b>직원 관리</b>, <b>근무표 관리</b>,
  <b>AI 기반 운영 지원</b> 기능을 제공하여 운영자가 보다 편리하게 매장을 관리할 수 있도록 설계하였습니다.
</p>

<p>
  실제 매장 운영 과정에서 발생하는 반복적인 관리 업무를 디지털화하여 업무 효율성을 높이고,
  운영 데이터를 체계적으로 관리하는 것을 목표로 개발한 팀 프로젝트입니다.
</p>


## 🌐 배포

## 📌 프로젝트 요약

### 1. 주제

매장 운영에 필요한 기능을 통합 관리할 수 있는 스마트 매장 관리 플랫폼

### 2. 목표

* 매출, 메뉴, 직원, 근무표 등 매장 운영 업무를 하나의 서비스에서 효율적으로 관리할 수 있도록 하는 것
* 운영 데이터를 시각화하여 매장 현황을 한눈에 파악할 수 있도록 하는 것
* 직원 관리 및 근무표 관리를 통해 체계적인 인력 운영 환경을 제공하는 것
* 반복적인 관리 업무를 디지털화하여 운영 효율성을 높이는 것

### 3. 핵심 기능

* 대시보드를 통한 매장 운영 현황 및 주요 데이터 확인
* 일별·주별·월별 매출 관리 기능
* 메뉴 등록, 수정, 삭제 및 메뉴 정보 관리
* 직원 등록, 수정, 삭제(CRUD) 기능
* 직원별 근무 일정 관리 및 주간 근무표 제공
* 직원 근무 시간 기반 인건비 계산 기능
* AI 기반 매장 운영 지원 기능
* 다양한 디바이스에서 사용할 수 있는 반응형 UI

### 4. 주요 기능

#### 🏠 Home

- AI 챗봇을 통해 매장 운영과 관련된 질문 및 데이터 조회 기능 제공
- 예상 매출, 예상 인건비, 부족 재고, 폐기 위험 품목 등 주요 운영 지표 확인
- 대시보드를 통한 실시간 매장 현황 모니터링 기능 제공
- 주간 매출 그래프를 통한 매출 추이 분석 기능 제공
- 발주 추천 및 운영 데이터를 기반으로 효율적인 의사결정 지원

#### 💰 Sales Management

* 일별 · 주별 · 월별 매출 조회
* 매출 데이터 분석
* 매출 현황 관리

#### 🍽️ Menu Management

* 메뉴 등록
* 메뉴 수정 및 삭제
* 메뉴 정보 관리

#### 👥 Staff Management

* 직원 등록, 수정, 삭제
* 직원 정보 관리
* 파트 및 시급 관리

#### 📅 Schedule Management

* 주간 근무표 관리
* 직원별 근무 일정 확인
* 직원 정보 연동

#### 🤖 AI Support

* AI 기반 매장 운영 지원
* 운영 데이터 활용 기능 제공

# 🛠 트러블 슈팅

## 👥 직원 관리 - 직원 데이터 식별 문제

### 문제 상황

직원 데이터가 배열 형태로 저장되면서 각 직원을 구분할 수 있는 고유 식별자(`_id`)가 없어 수정 및 삭제 시 특정 직원 데이터를 정확하게 구분하기 어려운 문제가 발생하였습니다.

특히 index 기반으로 데이터를 처리할 경우 정렬, 검색, 삭제 과정에서 데이터 불일치가 발생할 수 있었습니다.

### 해결 방법

직원 추가 시 `POST` 요청으로 기존 직원 배열에 데이터를 누적하고, 수정 및 삭제 시에는 `PUT` 요청을 통해 변경된 전체 직원 배열을 다시 저장하는 방식으로 개선하였습니다.

```js
// 직원 추가
{ $push: { employees: body.employees } }

// 직원 수정 / 삭제
{ $set: { employees: body.employees } }
```

### 결과

* 직원 데이터 수정 및 삭제 정확도 향상
* 데이터 식별 문제 개선
* 직원 목록과 근무표 데이터 관리 안정성 확보

## 📅 근무표 관리 - AI 분석 데이터 계산 오류

### 문제 상황

근무표 데이터를 기반으로 AI 분석을 진행하는 과정에서 `NaN` 또는 잘못된 계산값이 발생하는 문제가 있었습니다.

시급 값이 비어 있거나, 근무 시간이 정상적으로 계산되지 않거나, 매출 데이터가 문자열로 전달될 경우 인건비 및 매출 계산 결과가 정확하지 않았습니다.

### 해결 방법

데이터 계산 과정에서 예외 처리와 숫자 변환 로직을 추가하였습니다.

```js
// 시급이 없을 경우 0으로 처리
const pay = hours * (emp.hourlyWage || 0);

// 매출 데이터를 숫자로 변환하여 합산
const totalSales = (salesData || []).reduce((acc, cur) => {
  return acc + Number(cur.dailySales || 0);
}, 0);
```

### 결과

* `NaN` 오류 제거
* 인건비 및 매출 계산 정확도 향상
* AI 분석 결과의 안정성 개선
* 근무표 기반 데이터 처리 신뢰도 향상

## 💰 인건비 계산 - 야간 근무 및 급여 단위 계산 문제

### 문제 상황

직원별 근무 시간을 기준으로 인건비를 계산하는 과정에서 근무 종료 시간이 시작 시간보다 빠른 경우 계산값이 음수로 나오는 문제가 발생하였습니다.

예를 들어 `22:00-02:00`처럼 자정을 넘기는 야간 근무의 경우 단순히 종료 시간에서 시작 시간을 빼면 음수 시간이 계산되어 인건비가 올바르게 표시되지 않아 인건비가 정확하게 집계되지 않았습니다.

또한 일급, 주급, 월급을 각각 확인할 수 있어야 했기 때문에 선택한 급여 단위에 따라 계산 방식을 다르게 적용해야 했습니다.

### 해결 방법

근무 시작 시간과 종료 시간을 분 단위로 변환하여 차이를 계산하고, 계산 결과가 음수일 경우 24시간을 더해 자정을 넘긴 근무 시간도 정상적으로 계산되도록 처리하였습니다.

또한 `payType` 상태값을 기준으로 일급, 주급, 월급 계산식을 분리하여 사용자가 선택한 기준에 맞는 인건비를 확인할 수 있도록 구현하였습니다.

```js
let diff = toMin(end) - toMin(start);

if (diff < 0) {
  diff += 24 * 60;
}

return diff / 60;
```

### 결과

* `22:00 ~ 02:00`와 같은 야간 근무도 정상적으로 4시간으로 계산되도록 개선하였습니다.
* 근무 시간 계산 오류로 인해 발생하던 음수 인건비 문제를 해결하였습니다.
* 사용자가 일급, 주급, 월급 기준을 선택하면 즉시 해당 기준의 인건비를 확인할 수 있도록 구현하여 사용성을 향상시켰습니다.
* 직원별 근무 시간과 인건비가 정확하게 연동되어 보다 신뢰성 있는 급여 관리가 가능해졌습니다.
